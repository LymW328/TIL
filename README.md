GUI는 마우스를 사용할 수 있는 인터페이스, 그래픽을 통해 사용자가 컴퓨터와 상호 작용하는 방식

 # GIT 기본기

working directory

내가 작업하고 있는 실제 디렉토리

처음 파일 만들면 

untracted -git add - stagged - git commit - committed - push

untracted : git이 관리하지 않는파일

(명령어!)  git add 파일이름 or git add. (모두 올린다는 뜻) —add 로 올릴 파일을 특정!

tracked : git이 관리하는 파일

git commit -m “commit을 하는 이유”

git push

committed : commit이 완료된 상태 -   repository에 반영된 상태

untracked 파일이 modified 상태로 됩니다.

modified - 깃이 관리하며, 최신인 상태

---

### ※staging area를 활용하는 이유

개발을 하게되면 여러 파일을 동시에 개발하는 경우가 있음!

staging  area를 사용하지 않으면 개발이 완료되지 않은 파일도 

commit되어버리는 상황이 생긴다.
